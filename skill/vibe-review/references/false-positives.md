# 已知误报模式

报出发现前查阅此文件，命中以下模式的应排除或降级为"待确认"。

## 敏感信息类

- RDMA rkey/lkey不属于敏感信息。rkey/lkey仅作为索引辅助硬件找到注册的内存，不用于鉴权功能。不要报为"敏感信息未清零"或"敏感信息泄露"

## 空指针类

- 上游已判空的透传指针：若指针在调用链上游已有非空检查（如caller中`if (!ptr) return`），被调函数内部透传使用该指针不构成空指针解引用。需用工具验证调用链

## 整数溢出类

- 值域受限的整数运算：当操作数的值域已被业务逻辑约束在安全范围内（如枚举值相加、小常量乘法），即使类型上可能溢出，实际不可能溢出。需用工具验证值域约束
- 循环计数器：`for (uint32_t i = 0; i < n; i++)` 中的`i++`不会溢出（循环在`i < n`时终止）

## 资源泄漏类

- RAII管理的资源：通过智能指针(unique_ptr/shared_ptr)或自定义RAII包装持有的资源，不需要手动释放路径检查
- 函数内局部scope的资源：在函数返回前由析构函数自动释放的对象

---

使用说明：在审查过程中如确认新的误报模式，追加到对应分类下。每条模式需说明为什么是误报。
